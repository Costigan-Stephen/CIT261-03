We can do all of those things with JavaScript too, we can change the CSS of that object using JavaScript and flip flop between different types of JavaScript. The next example is of that, flip flopping between two different styles, or classes, when the user clicks on the object.

JavaScript - text.js

function change(x) {
	
	if (x.className == "text1"){
		x.className = "text2";
	}else if(x.className == "text2"){
		x.className = "text1";
	}
}

Lesson 8

Creating CSS3 Transitions and Animations in CSS and triggering them with JavaScript

Let us first look into Triggers and Transitions. The code you are about to look at is for a simple button that changes color each time it is pressed and when the mouse is hovered/not hovered over it. The button triggers a change in the CSS class of the object back and forth and could just as easily be tied into an animation if we feel like it. To keep it simple, we will post the animation after this. You will notice the javaScript is similar to the example in the previous lesson. Turns out it's a good simple way to flip flop styles.

Click On Me!

Goosebumps! That was fun. Now let's move on to Animations, which are just as fun! First, we need to cover a basic concept, that of KeyFrames. KeyFrames are basically motion points for an object, which make the animation process a lot easier because we don't need to design every single frame of the object. We specify a start point, some optional middle points, and an end point and CSS does the rest. Pretty nifty really. This next one was a lot of fun, it is literally a sphere that runs away from your mouse! Cool right? Let's look at an example!

JavaScript - sphere.js

function move(x) {
	
	if (x.className == "circle transform"){
		x.className = "circle transform-active";
	}else if(x.className == "circle transform-active"){
		x.className = "circle transform";
	}
}

Lessons 11-13

Designing, Defining, and Triggering CSS3 Transitions, Transforms, and Animations without Custom Libraries

This lesson can be best understood by a few short scripts written for a few different outcomes. The first script is written to better look at all elements of the lesson except for Arrays and Associative Arrays, which we will cover hereafter. The script will ask the user to input a number that will be raised to the power of 2 and then display all iterations of that calculation until we reach the answer. This is done so that the repetitions can be seen and understood as they happen rather than simply assuming they were or having to follow them in a script debugger.

In the spirit of simplification, we will look at the scripts as two different elements, JavaScript and HTML. The files can be combined by replacing the reference to an external script with the script itself, but for our purposes we will look at them separately. If you look at the JavaScript function, to which there is only one in this JavaScript, you will notice the condition asking if the input (t1) is greater than 0. If the statement is true, it will begin a while loop, which could just as easily be substituted for a For loop , but I like While loops and so this was done with that. The loop uses a temporary variable of i to increment each time until it is the same as t1 and then it quits.

JavaScript - (PowersOf2.js)

function calc() {

   	/* 		Defining Table
	* Input: 		None.
	* Processing: 	Determine all values between 2^0 to 2^31.
	* Output: 		Display result of calculation.
	*/     
	var cal = 0;
	var i = 0;
	var output = "";
	var t1 = document.getElementById('txt1').value;
	t1 = parseFloat(t1);
	
	//Only run when t1 > 0
	if (t1 > 0){
		while(i <= t1){
			cal = Math.pow(2, i);
			output += "2^" + i + " = " + cal + "
";
			i += 1;
		}
	}

	document.getElementById('outputDiv').innerHTML = output;
}
HTML

<!DOCTYPE HTML>
<html lang="en-us">
<head>
<meta charset="utf-8">
<title>Powers of Two and Binary</title>
<script src="PowersOf2.js"></script>
</head>

<body>
	<h1>Powers Of Two</h1>
    <p>
	What is 2^<input type="text" id="txt1" size="3">? <button type="button" onclick="calc()">GO</button>
	</p>
	<div id="outputDiv"></div>
</body>
</html>
