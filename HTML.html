Lesson 5

Local Storage API, Storing and Retrieving Simple Data, Arrays, Associative Arrays, and Objects

So there's some good fun to be had with Local Storage API's, but first we need to understand what they are. There are primarily 2 variations of that storage. First, Local Storage (obviously), secondly Session Storage. Both Local Storage and Session Storage serve different purposes and can store up to 5 MB of text. That may not seem like much, but when you consider that 1 character is equal to 1 byte, you realize that 5MB is about 5 Million characters. When you consider that the average book has about 350 words a page (1500 to 1800 characters per page), that's only about 400,000 - 500,000 characters (bytes), on average, which is about a tenth of the storage that either Local or Session storage provide.

So, what is Session Storage, and how does it differ from Local Storage? Simply put, Session storage is information that is stored only during your current session of browsing. This could be something like your shopping cart on a website or field entries when registering for something. Local storage, on the other hand, is information that exists beyond your current session. This can be just about anything, but a good example might be the chrome function that auto fills your information or remembers your passwords when you allow it to.

It's important to know that the commands for Local Storage and Session Storage are completely interchangeable, which is nice, so as we go through I will introduce both of them, but focus entirely on only one of them for the sake of simplicity. It's also worth noting that this will tie in to some more JSON, so make sure you check that out first! So what are the commands? Let's have a look!


(replace Local for Session if you need Session Storage)

//Storing Data methods 
1. localStorage.setItem("key", "value");
2. localStorage["key"] = value; 

//Example:
1. localStorage.setItem("Make", "Nissan");
2. localStorage["Make"] = "Nissan"; 

//Retrieving Stored Data
var myData = localStorage["key"];

//Example
var Make = localStorage["Make"]; //Not Nissan, notice the difference

//Removing Stored Data
localStorage.clear(); //Removes EVERYTHING!  Not advisable
localStorage.removeItem("key");

You can also store strings using Stringify, just so we are clear.

I know I said it in there, but I need to reiterate that localStorage.clear is NOT recommended. It's known as a scorched earth approach for a reason because it removes all stored local files, even ones you may not have wanted to remove. Also worth noting, you can store a string of variables using JSON Stringify, or you can store just one value, whatever you desire. For the purposes of our examples here we will only be storing individual values.

Now that we know the commands, let's mess with it a little bit. What do you say?

JavaScript - (storage.js)

function Local() {
	var data = document.getElementById('input').value;
	localStorage.setItem("key", data); 
	
	var test = localStorage.getItem("key");
    document.getElementById('local').value = test;
}	
function Session() {
	var data = document.getElementById('input').value;
	sessionStorage.setItem("key", data); 
	
	var test = sessionStorage.getItem("key");
    document.getElementById('session').value = test;
}

Lesson 6

DOM Manipulation Using createElement, appendChild, insertBefore, removeChild, etc

Do you know what you are looking at right now? A DOM. The DOM of this page is pretty great right? objects that make up my layout are part of a structure, a Document Object Model. What you might not know though is that the DOM can be CHANGED! Woah right? But what exactly does it mean?


It means that we can reorganize the objects on the layout! Or more than that, make new ones or remove them entirely!


var cars {Make: "ford", Model: "falcon", Year: 2010, Mileage: 80000}

You may notice that each variable in the Array has this format X:"Y", this is where the array becomes no more a series of values, but a series of objects. The car is no longer comprised of {ford,falcon,2010,80000}, but of a Make, Model, Year, and Mileage that can be parsed as objects that are associated with the array and not simply as destinations in an array (such as cars[1] and cars[2]...).

So let's look at this in terms of a script. If we were to create a very simple script that took the values of the vehicle and determined if you needed to start saving for a new car, what would it look for? For arguments sake, lets say that regardless of vehicle Make or Model, the yardstick was the mileage of the vehicle. When the vehicle went over 100,000 miles it would alert the driver that they should start saving for a new car. Ideally you might want to start much sooner, but for our purposes of simplicity let's assume this is an acceptable time to do this. The script for such an event might look something like this:

JavaScript - (assocArray.js)

function miles(){
	
	var t1 = document.getElementById('txt1').value;
	var t2 = document.getElementById('txt2').value;
	var t3 = document.getElementById('txt3').value;
	var t4 = document.getElementById('txt4').value;
	
	if(t4.length > 0){
		t4 = parseFloat(t4);
	}
	var output;
		
	//Regular Array >> var cars = ["Ford", "Falcon", "2010", 80000];
	//var cars {Make: "ford", Model: "falcon", Year: 2010, Mileage: 80000}
	var cars = {Make: t1, Model: t2, Year: t3, Mileage: t4};
	
	if(cars.Mileage > 100000){
		output = "Wow, over 100,000 miles! You might want to start saving up for a new car";
	}else if(cars.Mileage.length == 0){
		output = "";
	}else{
		output = "Only " + cars.Mileage + " miles, You're still good";
	}
	
	document.getElementById('output').innerHTML = output;
}

Lesson 10

HTML5 Tags - Video, Audio, and Canvas

This lesson can be best understood by a few short scripts written for a few different outcomes. The first script is written to better look at all elements of the lesson except for Arrays and Associative Arrays, which we will cover hereafter. The script will ask the user to input a number that will be raised to the power of 2 and then display all iterations of that calculation until we reach the answer. This is done so that the repetitions can be seen and understood as they happen rather than simply assuming they were or having to follow them in a script debugger.

In the spirit of simplification, we will look at the scripts as two different elements, JavaScript and HTML. The files can be combined by replacing the reference to an external script with the script itself, but for our purposes we will look at them separately. If you look at the JavaScript function, to which there is only one in this JavaScript, you will notice the condition asking if the input (t1) is greater than 0. If the statement is true, it will begin a while loop, which could just as easily be substituted for a For loop , but I like While loops and so this was done with that. The loop uses a temporary variable of i to increment each time until it is the same as t1 and then it quits.

JavaScript - (PowersOf2.js)

function calc() {

   	/* 		Defining Table
	* Input: 		None.
	* Processing: 	Determine all values between 2^0 to 2^31.
	* Output: 		Display result of calculation.
	*/     
	var cal = 0;
	var i = 0;
	var output = "";
	var t1 = document.getElementById('txt1').value;
	t1 = parseFloat(t1);
	
	//Only run when t1 > 0
	if (t1 > 0){
		while(i <= t1){
			cal = Math.pow(2, i);
			output += "2^" + i + " = " + cal + "
";
			i += 1;
		}
	}

	document.getElementById('outputDiv').innerHTML = output;
}
