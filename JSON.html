Lesson 3

JSON Parse, Stringify

At first glance, JSON seems an unnecessary language that does not give as much as it gets, but you might be surprised to learn that JSON provides a vital and useful function as a go between that exchanges data to and from a server. Why is this useful you might wonder? It turns out that data received from a server is received as a string and therefore it us up to JSON to decode that for use with JavaScript and other applications.

JSON Parse quite literally means take a string and turn it into an array. A string received from a webserver will likely look a lot like this: 


'{ "make":"nissan", "model":pulsar, "year":"2001", "mileage":"120000"}'

I went with a car example as that is what I used in the JavaScript examples and it seemed only right to keep that going. This string of values means nothing to JavaScript as a value, but to JSON it might as well be an Associative Array with the properties of Make, Model, Year, and Mileage. If we were to convert this, it would be as simple as this: 


var car = JSON.parse('{ "make":"nissan", "model":pulsar, "year":"2001", "mileage":"120000"}');

After it has been turned into an Array, we can deal with it as we would any other Array. If we wanted to make changes and send that information right back to the server, we would do something similar known as Stringify, which is quite literally turning the array into a string. Turns out it's rather simple so let's give it a whirl! Let's make an array of values and Stringify them! The code would look something like this:

JavaScript - Stringify.js

function string() {

	var make = document.getElementById('txt1').value;
	var model = document.getElementById('txt2').value;
	var year = document.getElementById('txt3').value;
	var mileage = document.getElementById('txt4').value;
	
	var cars = {"Make":make,"Model":model,"Year":year,"Mileage":mileage}
	var string = JSON.stringify(cars)

	document.getElementById('outputDiv').innerHTML = string;
}

Lesson 4

Using XML HTTP Request to Consume a JSON Web Service

This is where JSON starts to get a little more complicated. The XML HTTP Request is used to refresh information without refreshing a page. This might be useful in a lot of different ways, but a particularly useful one might be stock lists on sites such as Amazon, where a product might have limited stocks and which requires the ability to reload stock quantities without refreshing your session every few seconds. Such an action would not only slow your system, but take away from your experience.

One way that JSON can do this is by storing that information in an easily accessible place. That data can also be modified relatively simply much like you would with local storage or session storage (see HTML Lesson 5). I won't go much more into that, but I will show a simple text file, as an example, that could be read by a JSON HTTP Request. You will notice we are continuing with cars for continuity, but also because online car sales are a possible avenue for this kind of request to be made.

If data like this can be read easily by a program then it means that there can be an external storing of data and theoretically an extensive database. Information stored will appear in similar format to this and will be readable by XML HTTP Requests. I won't go into that too heavily, but I want to have a little fun.
